# Enterprise Trigger → Feature Mapping

*Load this when building the demo plan to decide which enterprise features
to highlight. Map each trigger found in discovery to the features below.*

---

## Trigger Mapping Table

| Trigger | Features to show | What to explain | Demo suggestion |
|---------|-----------------|-----------------|-----------------|
| **Compliance / security** | SSO/SAML, Custom Roles, Advanced Request Logs | Governance, audit trail, access control granularity | Walk through role creation and permission scoping; show audit log |
| **Multi-channel delivery** | Live Content API, GROQ, CDN | Single source of truth, deliver structured content anywhere | Query the same content for web and a second surface (mobile, email, signage) |
| **Content ops at scale** | Content Releases, Media Library, Canvas | Coordinated publishing across teams, asset management | Show a release with staged changes across multiple documents |
| **Developer productivity** | App SDK, Functions, Presentation Tool, schema-as-code | Build on the platform, not around it; version-controlled content model | Show schema in code, deploy, and demonstrate how changes flow |
| **Editor autonomy** | Studio customization, page builder patterns, Visual Editing | Editors work independently with a UI built for their workflow | Show a custom desk structure or page builder with no-code sections |
| **Migration from legacy CMS** | AI-assisted migration, structured content import, React/Next.js compatibility | Lower risk, faster time to value, incremental adoption path | Show a migration path sketch; discuss phased rollout |
| **AI and automation** | Content Agent, Functions, Agent API, MCP server | Contextual AI grounded in structured content, not bolted on | Show an automation triggered by content events; demonstrate agent context |

## Combining Triggers

Most enterprise prospects have 2-3 triggers, not just one. When combining:

- Lead the demo with the trigger closest to their primary pain point.
- Layer secondary triggers into the relevant section of the narrative arc
  (e.g., compliance features during the enterprise features section, not
  during the content model section).
- Don't show every enterprise feature. Show only what they asked about or
  what directly addresses a stated concern. Unrequested features dilute
  the demo.
